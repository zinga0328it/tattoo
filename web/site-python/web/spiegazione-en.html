How Our System Works

Our system uses an innovative architecture that combines Apache ProxyPass, Flask API, and Yggdrasil network to ensure maximum security and privacy for your data.

1. Public Frontend (Apache)

The website is hosted on a public Apache server with HTTPS certificate. This layer handles static content and user interface.

2. Secure Proxy (Apache ProxyPass)

When users interact with dynamic content (forms, API calls), Apache forwards requests to the isolated backend via Yggdrasil network.

3. Isolated Backend (Flask + SQLite)

The Flask application runs on a separate server accessible only through Yggdrasil. The database (SQLite/PostgreSQL) is completely isolated from the public internet.

üîí Security Features

  • Zero Public Exposure: Database and API are not accessible from traditional networks
  • Yggdrasil Encryption: All communications are encrypted end-to-end
  • Isolated Environment: Backend runs in a separate network segment
  • Minimal Attack Surface: Only essential ports are exposed
